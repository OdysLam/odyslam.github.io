Here comes Jekyll

How to build static sites in a matter of minutes

January 19, 2017 - 4 minute read -
web jekyll

I had to kick-start this blog with some information about this awesome platform.Jekyll helps you create simple sites in a fraction of the time, while giving you absolute freedom on how the site will be.This very blog is powered by Jekyll and was configured in a matter of hours.

But first things first, what is Jekyll?

It’s a simple static blog-aware website generator. It uses dynamic components like layouts and templates to create fully functional ready-to-host sites. Moreover it is written in Ruby, a quite new and kewl language that is especially famous for its web capabilities (Ruby on Rails).

At this point, someone could argue that wordpress does the same thing …well not quite:

Wordpress is a complete software suit where you can browse themes/templates/plugins, and apply them to your site, it’s quite easy but it also relies on back-end code and database management, so you can’t host the site to any static website server. Moreover, you have to maintain a Content Management System even for a simple blog or landing page.

On the other hand, as we can find at Jekyll’s site:

Jekyll is a simple, blog-aware, static site generator. It takes a template directory containing raw text files in various formats, runs it through a converter (like Markdown) and our Liquid renderer, and spits out a complete, ready-to-publish static website suitable for serving with your favourite web server. Jekyll also happens to be the engine behind GitHub Pages, which means you can use Jekyll to host your project’s page, blog, or website from GitHub’s servers for free.

So, in order to use Jekyll we need to configure a theme. A theme is basically a set of special files(html-css-js) that will serve as the basis for each and every page (called templates).

Example Theme:

I say special, because they incorporate Jekyll variables and other elements that make them to adjust automatically based on the settings in the YAML configuration file.

The YAML file is the cornerstone of each site. Each theme has its own and its content varies, according to what the theme can offer. In this file you simply fill in your site’s details (such is base_url, social media accounts, etc) and Jekyll takes care of everything else.

Now, you either need to know web-programming and create your own theme or just use a finished free jekyll theme created by someone else. You can find tons here, here and here.

Each theme has instructions on how to install and configure it. Νοte that simpler themes are less possible to encounter problems during installation.

Also simple themes are easier to fiddle with and change them according to your liking.The more you know about programming, the more you will be able to change the theme and custom-tailor it to your needs.

Lastly, you need to populate the posts folder that lives in each site directory with markdown files that Jekyll will turn to html.

When I created my first blog, in order to make a change, I literally changed lines of code and see what happened. Inarguably that was quite stupid and time-consuming. If you want to change stuff, please just head over to codeacademy and learn some basic html/css.

For Unix-based systems (macOS, linux, BSD)

After you have found the desirable theme, please head over to the docs and follow the instructions in order to install and configure Jekyll to your system. It’s quite simple and you could have your site up and running in a couple of hours (since it’s the first time).

For Windows

Although there is a guide for windows here, coding in windows is not the best idea since most things don’t run natively and you will need extra steps.I strongly suggest to use a Unix based systems. Especially for those who want to get into programming, linux is a must.
