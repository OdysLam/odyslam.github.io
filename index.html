Let's get into (Monty) Python

The script:

As I mentioned before, we are going to start with examining the python code, later we will move to javascript and html.

You can find the full code uploaded to github here

But let’s take things from the start:

Firstly, I do my imports:

As you can see I import a number of libraries, I am going to get into detail in later posts so as to keep this post nice and clean. I will explain where each and every library that is neaded.

The next important part is where I define the pins I am going to use. This way, I can easily change the pins by just changing the variable.

Here is the GPIO pinout again, you can see the corresponding number of each pin as well.

As I said before, we have 3 basic functions that are defined by webiopi: loop-setup-destroy. Loop is self explanatory, setup is run once when the webiopi service starts and detroy when we shut it down.

Setup:

First good programming advice is to always include debuging output functions so as you can monitor the program when you run it in debuging mode.

Now, as far as webiopi is concerned, the GPIO.setFunction(pin_number,type) it’s the first step, as it informs webiopi on what kind of pin each pin is . For example, here I have set we OutDoor pin to be a GPIO.OUT, in other words, a simple digital output (either 0v or 3.3v).

The servo pin must be set as pwm(pulse width modulation), for which you can read more about here. Webiopi has it’s own library to deal with servos, meaning that the needed pulse will be generated by the library and we don’t need to code it(you can code it by just setting a loop for the pin to turn on/off rappidly).We are just going to call the appropriate webiopi function and set the needed angle.

  • CloseUdoor is the function responsible for locking the door.(more later)

  • GPIO.digitalWrite(a,b) produces an output at the “a” pin,the output can be either logic HIGH or LOW (3,3v or 0v)

Next we have some library-specific functions that as I have said will be covered later when we speak about the various libraries.

Some custom functions:

The first function that we are going to cover is an easy one:

Locking/Unlocking building’s door:

It’s job is to to unlock my building’s door. By setting the pin either HIGH or LOW I enable or not current through a simple relay. The door would normaly unlock as long as I press a button, instead of that, I have connected the appropriate wires to a relay that will play the button’s role.

In order to turn it off again (and stop unlocking the door) I use the threading.Timer(seconds,function) function that is defined along with it’s library threading here.

The wonderful thing about this timer it’s that not only it does not freeze our program while waiting for it to execute (like webiopi.sleep), but we can also stop the Timer with the appropriate function. This way we can make sure that accidently executing a command won’t end in overlapping Timers that will both run the function.

Although I have not used them in the current version, it’s an addition that has to be made.

Locking/Unlocking appartment’s door:

Ok, don’t panic, it’s actually fairly straight forward.

Firstly, I use a global variable named Servo so as to track whether my door is locked or unlocked. This way I can prevent the servo from trying turn the gears while it’s arleady locked and thus potentially damage it.

Then, I check wether it’s locked Servo==0 and if it is, I send an alert to my weaved app on my iphone.(more later)

Stop right there you criminal scum!

Before we go any further we need to understand some basic principles about servo because they tend to have an arcane behaviour.

As you have read already,the servo position is defined by the length of the pulse or in other words it’s duty cycle. Every servo has it’s own default settings. For example,mine works in the range of 500μs το 2500μs, meaning that 1500μs is the 0 degrees. Moreover, for every 45 degrees, either + or -, the duty cycle is + or - 500μs.

Webiopi’s servo library uses the futaba standard, meaning that 0 is defined as 1520μs with a 400μs per 45 degrees.

Thus, in my code, my servo will go from -90 to +90 degrees. This is very important, because depending on the servo you may need to adjust the code to from 0 to 180 degrees if for example your servo works in the area of (1500μs-3500μs).

So..

I want the servo to turn as smoothly as possible, to accomplish that, I use a loop to set the angle to a range of (-90,+90). I could just set it straight to +90, but it wouldn’t be that smooth and may cause problems with the lock.

Lastly, depending on the button I press on my interface, I run the function with a different argument which determines how much time it will take to run the function that locks the door again.(for example, if I am leaving, I just need a 10 sec window to open the door, exit my appartment and then close it so it locks again)

Cool, by now you should have started getting comfortable with the python code, in the next posts we will be covering more libraries and functions.

In webiopi we trust

GLaDOS Framework

Ok, you have your Raspberry up and running as it’s sits quietly somewhere in the house. Now let’s examine the software framework of GLaDOS.

The main library-framework we are going to use is called Webiopi and it’s a wonderful, yet quite hard and suprisingly undocumented, tool.

From the site: WebIOPi includes an HTTP server that provides both HTML resources and a REST API to control things. Your browser will first load a HTML file, then the included Javascript will make Asynchronous calls to the REST API to control and update the UI. This method is very efficient, because it don’t need to refresh and download the whole page.

That doesn’t seem that simple

Well, actually it’s not that hard if you have some basic knowledge on how websites and servers work. The most straight-forward approach is the following:

  • Script file that is run by the python server and handles all Raspberry pi’s actions.

  • Website that uses Javascript to call functions that are defined and run by the python script file. It means that all you have to do with js is to call the appropriate function in the script. using python as much as possible makes the project much easier.

  • Weaved IoT suite(3rd party program intergrated into webiopi) to be able to access the Rpi server(where the site is hosted) outside of the lan (without port-forwarding ,etc).

As you can see from the image, you can control the gpios using directly Javascript functions. It’s both impractical and hard for beginners. We are going to exclusively python functions that will be called from Javascript(i.e pressing a button,executes python function “A”).

Important: It works both on Rpi 1 and 2, it just needs an update-fix and it works like a charm(going into detail later).

Important: In this simple example, there are only a couple of lines of css/javascript so it’s easy to put them in the .html file. It’s heavily advised that you create different folders and store each script in it’s own file. It’s a VERY good programming practice(some would say necessary).

There is a specific folder for your main python script and your index.html file, aka the mainpage.

Now, try to test the tutorial, it’s a very good first test to see how this particular library works.I am not going to explain all the code in detail, just some parts that I believe are important and/or hard to understand.

That setup-loop-destroy thingy looks familliar?

Yeah! Webiopi uses the same 3 function core as arduino to facilitate us. We are definately going to use the setup function. Loop is kinda optional since I prefer to create my own loops whenever that’s necessary, so as to keep the stress to the cpu as low as possible.(we need this project to be stable af)

That Javascript code seems strange

That because it is, it’s an old javascript library inside the webiopi which can be used to create very simple, but extremely ugly, buttons and slides that by default can control various gpios. It’s of no use as we going to control the gpio through python functions that are going to be called from the javascript, that way we can use whatever element we want, it’s customisable 100%.

Webiopi.sleep?

Well, that’s a way to slow-down loops. I prefer using the threading.timer which is part of the threading library. This way while you wait for the timer , the script can still execute commands in contrast with the webiopi.sleep function that totally freezes our script for the specified time.

Also, it houses a shit-ton of libraries, which means that it’s very easy to do a lot of different things since others have done all the dirty programming work for you.

As far as our project is concerned,for any possible upgrade or addition to the project that I have came up with (and they are a lot!) I still haven’t been able to find the appropriate library.

Also, as most of the interface objects(buttons,menus, etc) can be found ready online, it’s important that you can read a piece of code, understand it and then take the part you want to apply to our interface.

Also visit the appropriate course in Code Academy, it’s really worth the time.

Javascript

GlaDOS

Let’s get into detail

In this post I am going to explain thoroughly all the parts of the project so as you have an idea of what to come

GLaDOS it’s just the name I have given to my project since she is one of my favorite villains.It would be wise to avoid building the actual GLaDOS since it might cause severe pain,or even death.

Finally! What does it do? (°ロ°)☝

As I have said a thousand times it’s a smart-home project which aims at automating and facilitating a number of tasks, everything is controlled from a central web-page that is run locally on the Raspberry pi from a python server.

Tasks:

  • Lock/Unlock both the buildings door and the appartment’s. The first is done with the help of a relay and the second with the help of a servo and some 3D printed gears.

Disclaimer: I was still testing the script, that’s why I had set the lock/unlock time 2 sec.Now it’s automatically locks after 10-30-60 seconds depending on what button I press.

  • Control of 3 RF remote suko switches, by broadcasting a specific binary code I can turn on/off each switch remotely.

  • Control of Boiler. A power relay that uses AC as control signal is connected to an RF remote suko switch, same function as above. Bonus points for a software timer to fix the “Ooooh SHIT I forgot to turn off the boiler again”.

  • Remotely turn-on my computer by magic (packet).

Software:

To create the Interface, I built a simple .html page using html-css-javascript. In javascript I used the Jquery and webiopi libraries aswell.

To complete the tasks in the script file I use a number of python libraries (or the python library version of programs), namely:

  1. Webiopi
  2. wakeonlan
  3. vncdotool
  4. Pi_switch

Which we will get into detail when we examine the python code later in the guide.
