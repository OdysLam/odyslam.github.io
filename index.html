In webiopi we trust

GLaDOS Framework

Ok, you have your Raspberry up and running as it’s sits quietly somewhere in the house. Now let’s examine the software framework of GLaDOS.

The main library-framework we are going to use is called Webiopi and it’s a wonderful, yet quite hard and suprisingly undocumented, tool.

From the site: WebIOPi includes an HTTP server that provides both HTML resources and a REST API to control things. Your browser will first load a HTML file, then the included Javascript will make Asynchronous calls to the REST API to control and update the UI. This method is very efficient, because it don’t need to refresh and download the whole page.

That doesn’t seem that simple

Well, actually it’s not that hard if you have some basic knowledge on how websites and servers work. The most straight-forward approach is the following:

  • Script file that is run by the python server and handles all Raspberry pi’s actions.

  • Website that uses Javascript to call functions that are defined and run by the python script file. It means that all you have to do with js is to call the appropriate function in the script. using python as much as possible makes the project much easier.

  • Weaved IoT suite(3rd party program intergrated into webiopi) to be able to access the Rpi server(where the site is hosted) outside of the lan (without port-forwarding ,etc).

As you can see from the image, you can control the gpios using directly Javascript functions. It’s both impractical and hard for beginners. We are going to exclusively python functions that will be called from Javascript(i.e pressing a button,executes python function “A”).

Important: It works both on Rpi 1 and 2, it just needs an update-fix and it works like a charm(going into detail later).

Let’s get into the basics.

As you can see from this tutorial, the webiopi uses a certain skeleton structure:

home
- pi
  - myproject
    - python
    - html

Important: In this simple example, there are only a couple of lines of css/javascript so it’s easy to put them in the .html file. It’s heavily advised that you create different folders and store each script in it’s own file. It’s a VERY good programming practice(some would say necessary).

There is a specific folder for your main python script and your index.html file, aka the mainpage.

Now, try to test the tutorial, it’s a very good first test to see how this particular library works.I am not going to explain all the code in detail, just some parts that I believe are important and/or hard to understand.

That setup-loop-destroy thingy looks familliar?

Yeah! Webiopi uses the same 3 function core as arduino to facilitate us. We are definately going to use the setup function. Loop is kinda optional since I prefer to create my own loops whenever that’s necessary, so as to keep the stress to the cpu as low as possible.(we need this project to be stable af)

That Javascript code seems strange

That because it is, it’s an old javascript library inside the webiopi which can be used to create very simple, but extremely ugly, buttons and slides that by default can control various gpios. It’s of no use as we going to control the gpio through python functions that are going to be called from the javascript, that way we can use whatever element we want, it’s customisable 100%.

Webiopi.sleep?

Well, that’s a way to slow-down loops. I prefer using the threading.timer which is part of the threading library. This way while you wait for the timer , the script can still execute commands in contrst with the webiopi.sleep function that totally freezes our script.

In the next post we are going to see how to transfer data from our js script to our python server.

Are you talking to me?

Know your languages:

It’s important to know a set of programming languages in order to be able to understand the project.

Python

The most important is python, not only for this project, but for any raspberry pi project since it’s the main programming language. Python is a very easy-to-learn language as it has a very straight forward syntax ideal for beginners who just started learning programming. Also, it houses a shit-tone of libraries, which means that it’s very easy to do a lot of different things since others have done all the dirty programming work for you.

As far as our project is concerned,for any possible upgrade or addition to the project that I have came up with (and they are a lot!) I still haven’t been able to find the appropriate library.

I suggest you visit Code Academy to get a basic idea of how the language work.

HTML & CSS

They are the basis for any webpage, and in our case, the web interface is no exception. It’s important that you are able to write basic html/css code. Also, as most of the interface objects(buttons,menus, etc) can be found ready online, it’s important that you can read a piece of code, understand it and then take the part you want to apply to our interface.

Also visit the appropriate course in Code Academy, it’s really worth the time.

Javascript

By far the hardest language of them all, I tried to teach my-self from webiopi tutorials and random searching, it’s really not worth the time. You can cut the learning time by half if you visit again this course in Code Academy and learn the basics. I strongly recccomend that you do that before you dwelve into webiopi because of the lack of documentation.

Lastly

Keep in mind that the more you know about web front-end development, the more sofisticated your interface will be, both in terms of beauty and usability. Also, I personally believe that being able to create a simple landing page is very a very important tool no matter the field of studies.

GlaDOS

Let’s get into detail

In this post I am going to explain thoroughly all the parts of the project so as you have an idea of what to come

GLaDOS it’s just the name I have given to my project since she is one of my favorite villains.It would be wise to avoid building the actual GLaDOS since it might cause severe pain,or even death.

Finally! What does it do? (°ロ°)☝

As I have said a thousand times it’s a smart-home project which aims at automating and facilitating a number of tasks, everything is controlled from a central web-page that is run locally on the Raspberry pi from a python server.

Tasks:

  • Lock/Unlock both the buildings door and the appartment’s. The first is done with the help of a relay and the second with the help of a servo and some 3D printed gears.

Disclaimer: I was still testing the script, that’s why I had set the lock/unlock time 2 sec.Now it’s automatically locks after 10-30-60 seconds depending on what button I press.

  • Control of 3 RF remote suko switches, by broadcasting a specific binary code I can turn on/off each switch remotely.

  • Control of Boiler. A power relay that uses AC as control signal is connected to an RF remote suko switch, same function as above. Bonus points for a software timer to fix the “Ooooh SHIT I forgot to turn off the boiler again”.

  • Remotely turn-on my computer by magic (packet).

Software:

To create the Interface, I built a simple .html page using html-css-javascript. In javascript I used the Jquery and webiopi libraries aswell.

To complete the tasks in the script file I use a number of python libraries (or the python library version of programs), namely:

  1. Webiopi
  2. wakeonlan
  3. vncdotool
  4. Pi_switch

Which we will get into detail when we examine the python code later in the guide.

Raspberry pi 2

Hello, it’s me, Raspberry!

As you propably arleady know, Rpi is a small computer running a version of linux called Debian.Although it can run several more versions of linux, including custom ones that are made for specific purposes (i.e Voice Control), or Windows 10, Debian is by far the most common and versatile one.

We are going to cover the setup later in the blog.

It houses a wealth of hardware for a fairly limited price. Ranging from a camera display interface to 40 GPIO pin interface that transform rpi from a simple micro-computer to a versatile controler of electronic components.

More specifically:

  • 4 USB ports
  • 40 GPIO pins
  • Full HDMI port
  • Ethernet port
  • Combined 3.5mm audio jack and composite video
  • Camera interface (CSI)
  • Display interface (DSI)
  • Micro SD card slot
  • VideoCore IV 3D graphics core

I am not going to explain more at this point as it’s better to explain the various capabilities with examples from my project. It’s worth though to mention the pinout diagram as it’s very important for any project.

But what are those pesky GPIO anyhow?

That’s a fair question, GPIO stands for “General Purpose Input Output” meaning that these pins can be used as digital inputs or digital outputs. There are also pins that act as power supply(providing either 5v or 3.3v) or as ground. Some GPIO also have special functions (like pulse width modulation)

REMEMBER: Rpi use 3.3v as logical 1 and 0v as logical 0, unlike other microcontrolers,like Arduino which use 5v.

GPIO control?

As you propably have guessed arleady,a gpio can be controlled through scripts in various languages that use specific libraries that do most of the dirty work. My personal favorite is python as it’s easily understandable and ridiculously versatile. For these reasonons our main focus will be python although we will need some linux scripting as well. For the web interface we will need some basic html/css/javascript.

That’s all folks

I think that you have the all the neaded material to understand the basics of the raspberry pi 2 next will follow an overview of the project and slowly we will start geting into the software side of it.
