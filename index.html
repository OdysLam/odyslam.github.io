In this post I am going to explain thoroughly all the parts of the project so as you have an idea of what to come

Finally! What does it do? (°ロ°)☝

As I have said a thousand times it’s a smart-home project which aims at automating and facilitating a number of tasks, everything is controlled from a central web-page that is run locally on the Raspberry pi from a python server.

Tasks:

  • Lock/Unlock of both the buildings door and the appartment’s. The first is done with the help of a releay and the second with the help of a servo and some 3D printed gears.

  • Control of 3 RF remote suko switches, by broadcasting a specific binary code I can turn on/off each switch remotely.

  • Control of Boilder. A power relay that uses AC as control signal is connected to an RF remote suko switch, same function as above. Bonus points for a software timer to fix the “Ooooh SHIT I forgot to turn off the boiler again”.

*Remote turn on of my computer by magic (packet).

Software:

To create the Interface, I built a simple ‘.html’ page using html-css-javascript. In javascript I used the Jquery and webiopi libraries aswell.

To complete the tasks in the script file I use a number of python libraries (or the python library version of programs), namely:

  1. Webiopi
  2. wakeonlan
  3. vncdotool
  4. Pi_switch

Which we will get into detail when we examine the python code later in the guide.

Setting up the Raspberry

Let’s Roll

First things first, follow this guide:Noob Guide. and install an OS to the Raspberry.

Now that you have your system up and running let’s make our life a bit easier.

It’s not easy to have rpi always connected to a screen and a keyboard, we are going to run it headless. It means that we are going to use it’s terminal through the internet, from our computer. All we need is for the rpi to be connected to the internet and we also need to know it’s ip address.

  1. Connect Rpi to the internet, either via ethernet or wi-fi dongle

  2. Since it’s not cool to search and find the ip adress each time you want to fiddle with rpi, let’s make it have static ip according to this guide

  3. Configure ssh to gain command line control over the internet,using either command line in linux/OS X or a ssh client for Windows.

  4. To gain easy access to your files and folders in rpi from your pc, download filezilla to your computer and thenconfigure it for Rpi. it’s much easier to manipulate folders and files from this graphic design than from linux command line.

  5. Optional: Check this page as it contains a lot of useful commands. Thankfuly after some hours of Rpi experience you will start memorising the various commands, remember that command line is A-Z in Unix operating systems,like Lynux a.k.a Raspbian.

So, now that..

Raspberry pi 2

Hello, it’s me, Raspberry!

As you propably arleady know, Rpi is a small computer running a version of linux called Debian.Although it can run several more versions of linux, including custom ones that are made for specific purposes (i.e Voice Control), or Windows 10, Debian is by far the most common and versatile one.

We are going to cover the setup later in the blog.

It houses a wealth of hardware for a fairly limited price. Ranging from a camera display interface to 40 GPIO pin interface that transform rpi from a simple micro-computer to a versatile controler of electronic components.

More specifically:

  • 4 USB ports
  • 40 GPIO pins
  • Full HDMI port
  • Ethernet port
  • Combined 3.5mm audio jack and composite video
  • Camera interface (CSI)
  • Display interface (DSI)
  • Micro SD card slot
  • VideoCore IV 3D graphics core

I am not going to explain more at this point as it’s better to explain the various capabilities with examples from my project. It’s worth though to mention the pinout diagram as it’s very important for any project.

But what are those pesky GPIO anyhow?

That’s a fair question, GPIO stands for “General Purpose Input Output” meaning that these pins can be used as digital inputs or digital outputs. There are also pins that act as power supply(providing either 5v or 3.3v) or as ground. Some GPIO also have special functions (like pulse width modulation)

REMEMBER: Rpi use 3.3v as logical 1 and 0v as logical 0, unlike other microcontrolers,like Arduino which use 5v.

GPIO control?

As you propably have guessed arleady,a gpio can be controlled through scripts in various languages that use specific libraries that do most of the dirty work. My personal favorite is python as it’s easily understandable and ridiculously versatile. For these reasonons our main focus will be python although we will need some linux scripting as well. For the web interface we will need some basic html/css/javascript.

That’s all folks

I think that you have the all the neaded material to understand the basics of the raspberry pi 2 next will follow an overview of the project and slowly we will start geting into the software side of it.
